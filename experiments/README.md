# Experiments

This folder is for active creative and technical experiments.

Use it for:
- reference-image tests
- character-consistency tests
- prompt experiments
- model comparisons
- failed or partial pipelines worth revisiting

Do not treat files here as production-ready ad systems.

## Purpose

The goal is visibility.

When we are testing a new workflow, we keep the prompts, reference assets, notes, and outputs
here so it is obvious:
- what we tried
- which model we used
- what worked
- what failed
- what should be productized later

## Current rule

If a workflow is exploratory, put it under `experiments/` first.

Only move it into a stable top-level production area after it proves:
- repeatable output quality
- acceptable character consistency
- clear prompt structure
- viable editing/output flow
